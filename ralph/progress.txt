## Codebase Patterns
- AudioManager uses long-lived nodes stored as properties (engine, road, nitro, pad, etc.) — start/stop/update triplets follow the same pattern
- playingState.onEnter/onExit are the canonical hooks for starting/stopping gameplay audio
- updateBeat() is called every frame in playingState.update; it schedules beats ahead using AudioContext.currentTime (look-ahead scheduling pattern)
- beat.beatIndex increments once per scheduled beat — use Math.floor(beatIndex / 4) % N for bar-level events
- All audio nodes must connect through AudioManager.masterGain for mute to work
- Pad oscillators use `osc.detune` (Web Audio API) to apply cents offset without touching frequency directly

## 2026-03-13 - US-001
