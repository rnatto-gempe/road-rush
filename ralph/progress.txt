## Codebase Patterns
- Single-file pure JS + HTML5 Canvas game in `src/main.js` (~2019 lines)
- FSM states are `const` objects with `onEnter()`, `update(dt)`, `render(ctx)` methods
- States reference each other via closure (forward references work since methods are called at runtime)
- Module-level variables hold game state (particles[], traffic[], gameState, etc.)
- `resetGameState()` must reset ALL module-level state introduced by new features
