## Codebase Patterns
- Item system follows a consistent pattern: constants → gameState array + nextSpawnDistance → spawn/update/render functions → bloom entry → update/render calls in playingState
- Spawn rarity is controlled by SPAWN_MIN/SPAWN_MAX constants (px traveled between spawns). Shield: 5000-7500px, Nitro: 2500-4000px
- All SFX use Web Audio API procedural synthesis in the AudioManager object — no audio files
- Collection uses AABB overlap between getPlayerHitbox() and item bounding box
- Float text feedback via spawnFloatText(x, y, text, color) on collection
- Bloom/glow added in renderBloom() function for each collectible type
- HUD bars (nitro, weapon) rendered at bottom-center of screen
- Bullet system: bullets[] array in gameState, updateBullets() handles spawn+move+collision, renderBullets() has phase-aware visuals
- Vehicle destruction uses spawnVehicleDestroyParticles() for fire+debris particle burst
- Bullet-vehicle collision uses same aabbOverlap() + getVehicleHitbox() as player collision
- Debug shortcuts in playingState: D=toggle debug, P=25k score, O=50k score, W=activate weapon
