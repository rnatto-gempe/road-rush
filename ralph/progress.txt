## Codebase Patterns
- AudioManager is a module-scope singleton in src/main.js — init() creates AudioContext, resume() handles autoplay policy
- All audio nodes must connect through AudioManager.masterGain for global mute support
- Use gain envelopes (0.01s attack/release) on all oscillators/noise to avoid clicks
- Use oscillator.onended / source.onended to disconnect nodes and prevent memory leaks
- AudioContext is created on first Enter press in titleState (browser autoplay policy compliance)
- The game has two places where Enter starts gameplay: titleState and gameOverState — both need audio handling
- FSM calls onExit() automatically if defined — use it for cleanup when leaving a state (covers all exit paths)
- Long-lived audio nodes (engine, road, nitro) use start/stop/update pattern: start in onEnter, stop in onExit, update per frame
- Nitro audio activation is driven by nitroTimer in playingState.update, not in updateNitroItems — keeps audio logic centralized
- Reuse noise buffers (store on AudioManager) when creating looping noise sources to avoid repeated buffer generation
- exponentialRampToValueAtTime cannot ramp to 0 — use 0.001 as the floor for all decay envelopes
- Shield break has TWO trigger locations: handleVehicleCollision and border collision — both need consistent SFX
- Pickup SFX gains (0.25-0.35) sit below collision/explosion hierarchy (0.4-0.8) — maintain this for new SFX
- Coin cluster ascending pitch uses AudioManager.coinSeq with a 0.4s timeout, decremented via updateCoinSeq(dt) in game loop
- Near miss/combo/bonus SFX gains (0.2-0.3) sit at or below pickup SFX hierarchy — maintain volume ordering: explosion > collision > pickups >= bonus SFX
- triggerNearMiss() is the single trigger point for near miss whoosh + combo stinger — no need to wire multiple locations
- Overtake bonus SFX is in the traffic filter callback (v.y > 760 && !v.collided) — fires frequently, keep it subtle
- Title drone and state-specific ambient audio: start in onEnter (guard with ctx check), stop in onExit — follows same pattern as engine/road
- Beat scheduler uses AudioContext.currentTime look-ahead pattern — schedule ~4 beats ahead, refill as time advances (never use setTimeout/setInterval for musical timing)
- State-specific ambient audio (rumble, drone) follows onEnter/onExit pattern — start in onEnter, stop in onExit; cancelScheduledValues before new ramps on stop
- Global HUD overlays (mute icon) render AFTER ctx.restore() in gameLoop, outside the shake transform, to stay stable in all states
