## Codebase Patterns
- AudioManager is a module-scope singleton in src/main.js — init() creates AudioContext, resume() handles autoplay policy
- All audio nodes must connect through AudioManager.masterGain for global mute support
- Use gain envelopes (0.01s attack/release) on all oscillators/noise to avoid clicks
- Use oscillator.onended / source.onended to disconnect nodes and prevent memory leaks
- AudioContext is created on first Enter press in titleState (browser autoplay policy compliance)
- The game has two places where Enter starts gameplay: titleState and gameOverState — both need audio handling
- FSM calls onExit() automatically if defined — use it for cleanup when leaving a state (covers all exit paths)
- Long-lived audio nodes (engine, road, nitro) use start/stop/update pattern: start in onEnter, stop in onExit, update per frame
- Nitro audio activation is driven by nitroTimer in playingState.update, not in updateNitroItems — keeps audio logic centralized
- Reuse noise buffers (store on AudioManager) when creating looping noise sources to avoid repeated buffer generation
