## Codebase Patterns
- `DIFFICULTY` config object at top of main.js is the single source of truth for all tunable difficulty params; constants below it alias DIFFICULTY values for backward compat
- `debugMode` (module-level bool) toggled by D key in playingState.update() via consumeKey; rendered in playingState.render()
- Speed ramp: phase 1 uses absolute formula `INITIAL + elapsed * RATE` (not incremental), phase 2 uses `speed += RATE2 * dt`; both capped with Math.min
- `updateScrollSpeed(dt)` called BEFORE `scrollOffset +=` and `distanceTraveled +=` so the new speed applies immediately
- Player object lives in `gameState.player`: {x, y, width, height, vx}
- `getPlayerHitbox(player)` returns {x, y, w, h} at 80% size (player.x+4, player.y+6.5, 32, 51)
- `updatePlayer(dt)` handles ArrowLeft/Right input, acceleration/deceleration, and road clamping
- `renderPlayer(ctx, player)` draws rounded red rect + windshield + rear window
- ctx.roundRect() is available in modern browsers (Chrome 99+, Safari 15.4+)
- All game code lives in `road-rush/src/main.js` (single file, vanilla JS)
- Road constants: ROAD_LEFT=40, ROAD_RIGHT=360, ROAD_WIDTH=320, LANE_WIDTH=80, LANE_COUNT=4
- `gameState` is a shared mutable object; `resetGameState()` resets it on retry
- FSM: `fsm.transition(state)` calls onExit/onEnter; each state has update(dt) and render(ctx)
- Input: `keys[key]` for held keys, `consumeKey(key)` for one-shot presses
- Game loop uses fixed timestep accumulator at 1/60s (FIXED_DT)
- scrollOffset scrolls indefinitely (no wrap needed in storage â€” only modulo at render time)
- Canvas context state (lineDash, lineWidth, etc.) must be reset after use to avoid bleed-over
- Traffic stored in `gameState.traffic[]` as {type, x, y, lane, width, height, ownSpeed, laneChangeTimer, laneChanging}
- `getVehicleHitbox(v)` returns 80% centered hitbox â€” reusable for all vehicle types
- Visual screen speed = scrollSpeed - ownSpeed (downward on screen); ownSpeed set at spawn time, not updated with ramp
- 4 vehicle types: sedan(blue,40x60,60%speed), truck(gray,60x72,40%speed), sports(darkred,36x58,85%speed,minTime=40s), moto(yellow,20x48,70%speed,minTime=60s)
- Spawn config: `getSpawnConfig(elapsed)` returns {maxCount, spawnMin, spawnMax}; type selection via `chooseVehicleType(elapsed)`
- Spawn tracking: `gameState.nextSpawnDistance` compared to `gameState.distanceTraveled`
- Fairness constraint: `isFairToSpawn(candidate)` checks 96px vertical bands (step 48px) for 60px free corridor; `getVehicleXExtent(v, 1.0)` projects lane-change extents; skip spawn (don't advance nextSpawnDistance) if constraint fails
- `resetGameState()` must clear `gameState.traffic = []` and reset `nextSpawnDistance`
- Graded collision: `speedPenaltyMultiplier` + `speedPenaltyTimer` (module-level) apply multiplicative slow; `getEffectiveScrollSpeed()` wraps `scrollSpeed * multiplier`; ramp writes base `scrollSpeed` independently
- Invulnerability: `invulnTimer` (module-level), blink via `Math.floor(invulnTimer / BLINK_INTERVAL) % 2` in renderPlayer
- Particle system: module-level `particles[]` array with `{x,y,vx,vy,life,maxLife,color,size}` objects; update/render separate functions
- Fuel system: `gameState.fuel` (0-100), drains per frame using `getEffectiveScrollSpeed()`; `gameState.fuelItems[]` with `{x,y,collectAnim}` (collectAnim=null until collected); road-fixed so scroll at effective speed (ownSpeed=0)
- collectAnim pattern: `{timer: 0.2}` on item, skip normal update while non-null, splice when timer<=0; use `1 - timer/duration` as progress for scale/alpha
- `gameOverState.causeOfDeath` must be set BEFORE `fsm.transition(gameOverState)`
